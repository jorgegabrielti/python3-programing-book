```python
#!/usr/bin/env python3

print("Hello", "World!")

```
A primeira linha é um comentário. No Python, os comentários começam com um # e continuam até o final da mesma linha. (Iremos explicar o comentário um tanto enigmático mais adiante.) A segunda linha está em branco fora de strings delimitadas, o Python ignora linhas em branco, mas elas são geralmente muito úteis para humanos quebrarem blocos de código muito grandes, tornando-os mais fáceis de se ler. A terceira linha é um código Python. Aqui, a função print() foi evocada com dois argumentos, cada um do tipo str (string [cadeia]; isto é, uma sequência de caracteres).
Cada comando encontrado em um arquivo .py é executado em turnos, começando com o primeiro e, progressivamente, linha por linha. Isso é diferente de algumas outras linguagens, por exemplo, C++ e Java, as quais possuem uma função particular ou um método de nome especial pelo qual elas iniciam. O fluxo de controle pode ser desviado, como veremos quando discutirmos as estruturas de controle do Python na próxima seção.
Iremos assumir que os usuários do Windows mantêm seus códigos Python no diretório C:\py3eg e que os usuários Unix, Linux e Mac OS X mantêm seus códigos no diretório $HOME/py3eg. Salve, então, hello.py dentro do diretório py3eg e feche o editor de texto.
Agora que já temos um programa, podemos executá-lo. Os programas Python são executados pelo interpretador do Python e, normalmente, isso é feito dentro de um terminal do Windows. No Windows, este terminal é chamado de "Prompt de Comando" ou "DOS Prompt" ou "MS-DOS Prompt", ou algo similar, e, em geral, está disponível em (iniciar) Start->All (todos os programas) Programs->(acessórios) Accessories. No Mac OS X, o terminal é fornecido através do programa Terminal.app (localizado por default em Aplications/Utilities), que também pode ser encontrado utilizando o Finder (Localizador) e, em outros Unixes, podemos usar um xterm ou um terminal que é fornecido pelo ambiente de janelas, por exemplo, konsole ou gnome-terminal.

No Unix, quando um programa é invocado no terminal, os primeiros dois bytes do arquivo são lidos.* Se estes bytes são os caracteres ASCII #!, o shell assume que o arquivo deve ser executado por um interpretador e que a primeira linha do arquivo especifica qual interpretador usar. Essa linha é chamada de linha *shebang* (execução shell) e, se presente, deverá ser a primeira linha do arquivo.
A linha shebang é comumente escrita em uma destas duas formas:
```text
#!/usr/bin/python3

```
ou:
```text
#!/usr/bin/env python3

```
Se a escrita usar a primeira forma, o interpretador especificado é usado. Essa forma pode ser necessária para os programas Python que serão executados por um servidor da web, ainda que o caminho especificado possa ser diferente do que foi mostrado. Se a escrita usar a segunda forma, o primeiro interpretador python3 encontrado no ambiente corrente do shell é usado. A segunda forma é mais versátil, porque permite a possibilidade de o interpretador Python 3 não ficar localizado em /usr/bin (por exemplo, ele poderá ficar em /usr/local/bin ou instalado em $HOME).
